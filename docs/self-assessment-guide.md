# ACP Self-Assessment Guide

This guide explains how organizations can apply the **Active Cyber Program (ACP)** framework independently to evaluate and improve their cybersecurity program.

The goal is to help organizations perform a **structured self-assessment** and prepare for a potential ACP certification.

---

# Overview

The ACP self-assessment consists of the following steps:

1. Define Scope
2. Identify Stakeholders
3. Collect Information
4. Perform Assessment
5. Gather Evidence
6. Score Results
7. Identify Gaps
8. Define Improvement Actions

---

# 1. Define Scope

Define what part of the organization will be assessed.

This may include:

• entire organization  
• specific business units  
• IT systems or environments  
• cloud or on-premise infrastructure  

A clearly defined scope ensures meaningful results.

---

# 2. Identify Stakeholders

Identify key stakeholders involved in cybersecurity.

Typical roles include:

• IT management  
• security responsible persons  
• system administrators  
• business representatives  

---

# 3. Collect Information

Gather relevant documentation and information.

Examples:

• policies and procedures  
• system documentation  
• architecture diagrams  
• security reports  

---

# 4. Perform Assessment

Use the ACP assessment questions to evaluate each domain.

For each control:

• review current implementation  
• discuss with relevant stakeholders  
• document findings  

---

# 5. Gather Evidence

Collect evidence supporting each assessment result.

Examples:

• policies  
• system configurations  
• screenshots  
• logs and reports  

Without evidence, controls should not be rated highly.

---

# 6. Score Results

Apply the ACP scoring model:

0 – Not Implemented  
1 – Partially Implemented  
2 – Implemented  
3 – Actively Managed  

Calculate:

• control scores  
• domain scores  
• overall ACP score  

---

# 7. Identify Gaps

Identify areas where:

• controls are missing  
• implementation is incomplete  
• risks are not addressed  

These gaps represent improvement opportunities.

---

# 8. Define Improvement Actions

Define concrete actions to improve cybersecurity.

Examples:

• implement missing controls  
• improve processes  
• introduce monitoring  
• strengthen governance  

---

# Optional: Prepare for ACP Certification

Organizations may use the self-assessment to prepare for an official ACP assessment.

Before requesting certification:

• ensure minimum requirements are met  
• collect complete evidence  
• review critical domains  

---

# Continuous Improvement

Cybersecurity is an ongoing process.

Organizations should:

• repeat assessments periodically  
• track improvements  
• update controls as needed  

---

# Summary

The ACP self-assessment enables organizations to:

• understand their cybersecurity maturity  
• identify weaknesses  
• improve security systematically  
• prepare for certification  

The ACP framework is designed to be practical, structured, and applicable to organizations of any size.
