# ACP Assessment Process

This document describes the standard process used to perform an **Active Cyber Program (ACP) Assessment**.

The assessment process is designed to ensure that evaluations are **structured, transparent, and consistent** across organizations.

The goal of the assessment is to determine whether an organization operates an **active cybersecurity program** according to the ACP framework.

---

# Overview of the ACP Assessment Process

The ACP assessment typically consists of the following phases:

1. Assessment Preparation
2. Scope Definition
3. Documentation Review
4. Interviews
5. Technical Validation
6. Control Evaluation
7. Reporting
8. Certification Decision

---

# 1. Assessment Preparation

During the preparation phase the assessment is planned and coordinated with the organization.

Typical activities include:

* initial contact with the organization
* explanation of the ACP framework
* agreement on assessment scope
* scheduling of assessment activities
* identification of key stakeholders

Organizations may receive guidance on how to prepare relevant documentation and evidence.

---

# 2. Scope Definition

The scope of the assessment defines which parts of the organization will be evaluated.

The scope may include:

* specific business units
* IT infrastructure
* cloud environments
* operational processes
* data protection practices

The scope must be clearly documented before the assessment begins.

---

# 3. Documentation Review

The assessor reviews relevant cybersecurity documentation provided by the organization.

Typical documents include:

* security policies
* risk management documentation
* system architecture documentation
* operational procedures
* incident response plans
* security reports

This phase helps the assessor understand the organization’s cybersecurity structure.

---

# 4. Interviews

Interviews are conducted with personnel responsible for cybersecurity and IT operations.

Possible interview participants include:

* IT management
* security officers
* system administrators
* operations personnel
* management representatives

Interviews help confirm how security processes work in practice.

---

# 5. Technical Validation

Technical validation may be performed to confirm the implementation of cybersecurity controls.

Examples include:

* demonstration of security systems
* review of system configurations
* inspection of monitoring tools
* verification of access controls
* review of backup processes

Technical validation ensures that controls are not only documented but operational.

---

# 6. Control Evaluation

The assessor evaluates the organization against the **ACP assessment checklist**.

Each control is scored according to the ACP scoring model.

Evaluation considers:

* documented processes
* operational practices
* technical implementation
* supporting evidence

Domain scores and overall scores are calculated based on the scoring model.

---

# 7. Assessment Report

After completing the evaluation, an **ACP Assessment Report** is prepared.

The report typically includes:

* assessment scope
* summary of cybersecurity posture
* domain scores
* maturity level
* identified strengths
* improvement recommendations

The report provides the organization with a clear overview of its cybersecurity program.

---

# 8. Certification Decision

Based on the assessment results, a certification decision is made.

Possible outcomes include:

ACP Certification Granted
ACP Certification Pending Improvements
ACP Certification Not Granted

Organizations that meet the certification requirements may receive the **Active Cyber Program (ACP) Certification**.

---

# Continuous Improvement

Organizations are encouraged to use the assessment results to strengthen their cybersecurity program.

Recommended actions may include:

* implementing improvement measures
* strengthening security governance
* improving monitoring and incident response
* increasing cybersecurity maturity

Organizations may request a **reassessment** after implementing improvements.

---

# Confidentiality

All information collected during the ACP assessment must be handled confidentially.

Assessors must protect sensitive organizational information and use it only for the purpose of the ACP evaluation.
