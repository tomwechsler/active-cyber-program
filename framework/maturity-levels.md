# ACP Maturity Levels

The **Active Cyber Program (ACP)** framework uses maturity levels to evaluate how effectively cybersecurity is implemented and actively managed within an organization.

While the ACP control domains define **what must be addressed**, the maturity levels describe **how well these controls are implemented and operated**.

The maturity model allows organizations to understand their current cybersecurity capabilities and identify opportunities for improvement.

---

# ACP Maturity Model

The ACP maturity model consists of five levels.

| Level   | Name      | Description                                                                     |
| ------- | --------- | ------------------------------------------------------------------------------- |
| Level 1 | Initial   | Cybersecurity activities are informal or reactive.                              |
| Level 2 | Managed   | Basic security controls are implemented but not consistently managed.           |
| Level 3 | Defined   | Security processes are documented and consistently applied.                     |
| Level 4 | Measured  | Security activities are monitored and measured.                                 |
| Level 5 | Optimized | Cybersecurity is continuously improved and integrated into business operations. |

---

# Level 1 – Initial

At this level cybersecurity practices are mostly reactive.

Security activities may occur but they are not formally structured or consistently implemented.

## Characteristics

* Security responsibilities are unclear
* Security controls are implemented ad hoc
* Documentation is limited or missing
* Risk management is informal
* Security monitoring is minimal

Organizations at this level are typically vulnerable to operational disruptions and security incidents.

---

# Level 2 – Managed

Basic cybersecurity controls exist and are partially implemented.

However, security practices may vary across systems or departments and are not yet fully standardized.

## Characteristics

* Some security policies exist
* Basic security technologies are deployed
* Responsibilities for security are defined
* Risk management activities occur occasionally
* Security incidents are handled but processes may not be documented

Organizations at this level have started to address cybersecurity but lack consistency.

---

# Level 3 – Defined

Cybersecurity processes are formally documented and implemented across the organization.

Security responsibilities and procedures are clearly defined and communicated.

## Characteristics

* Security policies and procedures are documented
* Risk management processes are established
* Security controls are consistently applied
* Security responsibilities are clearly assigned
* Incident response processes are defined

Organizations at this level demonstrate structured cybersecurity management.

---

# Level 4 – Measured

Cybersecurity activities are actively monitored and evaluated.

Security performance is measured using defined metrics and reporting mechanisms.

## Characteristics

* Security metrics and KPIs are defined
* Security monitoring and logging are implemented
* Security reviews are conducted regularly
* Security incidents are analyzed
* Leadership receives regular security reporting

Organizations at this level operate an **active cybersecurity program**.

---

# Level 5 – Optimized

Cybersecurity is fully integrated into organizational strategy and continuous improvement.

Security capabilities evolve based on threat intelligence, risk analysis, and operational experience.

## Characteristics

* Continuous security improvement processes exist
* Advanced monitoring and detection capabilities are implemented
* Security automation may be used
* Security awareness is embedded in the organization
* Cybersecurity is integrated into strategic decision making

Organizations at this level demonstrate a highly mature and resilient cybersecurity posture.

---

# Maturity Evaluation

During an ACP assessment, each control domain is evaluated and assigned a maturity level.

The overall maturity of the organization is determined by combining the maturity levels across all domains.

The maturity model helps organizations:

* understand their cybersecurity capability
* identify improvement areas
* measure progress over time
* strengthen their cybersecurity program

---

# Target Maturity for ACP Certification

Organizations seeking **Active Cyber Program (ACP) Certification** are expected to demonstrate cybersecurity maturity at least at **Level 3 – Defined** across the majority of domains.

Higher maturity levels indicate stronger and more actively managed cybersecurity programs.
