# ACP Evidence Requirements

This document describes the types of evidence that may be collected during an **Active Cyber Program (ACP) Assessment**.

Evidence helps verify that cybersecurity controls are **not only documented but actively implemented and managed** within the organization.

Assessors evaluate evidence through documentation review, interviews, and technical validation.

---

# Purpose of Evidence

Evidence is required to demonstrate that cybersecurity practices are operational and effective.

Evidence helps confirm that:

* security controls exist
* processes are implemented
* responsibilities are defined
* security activities are actively performed

Evidence should reflect the **current operational state** of the organization.

---

# Types of Evidence

Evidence may include several types of information sources.

## Documentation

Formal documents describing cybersecurity policies, procedures, or governance structures.

Examples:

* Information security policies
* Security procedures
* Risk management documentation
* Incident response plans
* Backup policies
* Access control policies

---

## Operational Records

Operational records demonstrate that cybersecurity activities are actively performed.

Examples:

* security monitoring reports
* incident response records
* patch management reports
* vulnerability scan results
* access review records
* backup execution logs

---

## Technical Configuration

Technical evidence confirms that security controls are implemented within systems and infrastructure.

Examples:

* firewall configuration
* identity management settings
* encryption configuration
* endpoint protection dashboards
* cloud security configuration
* monitoring system configuration

---

## Interviews

Interviews with responsible personnel may provide additional confirmation that security processes are actively performed.

Examples of interview participants:

* IT administrators
* security officers
* management representatives
* operations personnel

Interviews help validate how cybersecurity processes work in practice.

---

# Domain-Specific Evidence Examples

The following examples illustrate typical evidence for each ACP control domain.

---

## Business & Security Governance

Possible evidence:

* cybersecurity strategy
* security governance structure
* management reporting
* security planning documents

---

## Risk Management

Possible evidence:

* risk register
* risk assessment reports
* risk mitigation plans
* third-party risk evaluations

---

## Security Policies & Processes

Possible evidence:

* information security policy
* acceptable use policy
* incident response procedures
* backup and recovery policies

---

## Identity & Access Management

Possible evidence:

* user management procedures
* access review documentation
* multi-factor authentication configuration
* privileged access policies

---

## Infrastructure Security

Possible evidence:

* asset inventory
* patch management reports
* system configuration standards
* cloud security configuration

---

## Endpoint Security

Possible evidence:

* endpoint management dashboard
* antivirus or EDR status reports
* device inventory
* encryption status reports

---

## Network Security

Possible evidence:

* firewall rule sets
* network architecture diagrams
* VPN configuration
* wireless network security settings

---

## Data Protection & Encryption

Possible evidence:

* encryption policies
* backup procedures
* data classification guidelines
* secure file transfer processes
* encrypted archive usage (e.g. encrypted ZIP)

---

## Monitoring & Incident Response

Possible evidence:

* log monitoring reports
* SIEM dashboards
* incident response reports
* escalation procedures

---

## Continuous Improvement

Possible evidence:

* security audit reports
* vulnerability assessment results
* security training records
* improvement plans

---

# Evidence Validation

Assessors may validate evidence using multiple methods:

* documentation review
* technical verification
* system demonstrations
* operational walkthroughs

Evidence should be **sufficient to demonstrate active cybersecurity management**.

---

# Confidentiality

All evidence provided during an ACP assessment must be handled confidentially.

Sensitive organizational information must be protected and used solely for the purpose of the ACP assessment.
